# 2025-12-28 20:51 — `约定.md` 可读性与进度口径调整

## 背景

- 终端/部分 UI 在显示 `↔`、`✅` 等字符时可能会退化成 `?`，导致“约定文件”阅读体验变差、容易误解。
- 主仓库的 `plan.md` 已刷新为“下一步迭代计划”，但 `约定.md` 的“当前执行进度”仍沿用旧里程碑口径，容易造成混淆。

## 变更

- 将 `约定.md` 中的 `↔` / `✅` 等符号替换为 ASCII 表达（如 `<->`、`DONE`），提升跨平台可读性。
- 将“当前执行进度”拆成：
  - **基线能力（旧里程碑）**：标记为 `DONE`，便于回溯历史实现；
  - **下一阶段（新 plan.md）**：明确从 `plan.md` 的 M1 开始推进。
- 在文件顶部补充提示：目标项目（workspace）若需要约定文件，优先使用 `docs/templates/workspace_约定.md` 模板并按项目裁剪。

## 验证
